## DeepSeek-PC 桌面端项目构建

使用 grill-with-docs skill 完成了 DeepSeek 桌面端（Wails v3 + vanilla JS）的 grilling 环节和完整代码实现。

### 关键决策（记录于 CONTEXT.md + ADR）
- **窗口架构**: 完全分离的前端，主窗口加载 https://chat.deepseek.com，设置窗口加载本地资产
- **主窗口**: 标准窗口（非 Frameless），有标题栏和关闭/最小化/最大化按钮
- **设置 UI**: 标准 Wails 模式（Go Service + 前端页面），JSON 文件持久化
- **单例**: Wails v3 内置 `SingleInstanceOptions`，UniqueID=com.deepseek.desktop
- **热键**: 改用 `golang.design/x/hotkey` 系统级热键（替代 `app.KeyBinding`），窗口隐藏也能唤醒。使用 `RegisterAndSwap` 模式安全替换。主线程安全通过 `application.InvokeSync` 保证。
- **用户脚本**: `ExecJS()` 运行时注入，支持 startup/load 两种时机，不受 CSP 限制
- **快捷键 UI**: 参考 jPaste 的 SettingsPage.jsx + hotkey_windows.go 模式，但简化为单修饰键

### 技术备忘
- Wails v3 实际 API 与文档有出入: `SystemTray.New()` 无参数、`SetIcon([]byte)`、`NewService[T]` 泛型
- 绑定路径: `frontend/bindings/changeme/internal/settings/`，导出 Service/Data
- 绑定通过 `wails3 generate bindings` 生成，构建流程已有 `wails3 build` 自动完成
- 系统级热键用 `golang.design/x/hotkey`，回调在 goroutine 中，需用 `application.InvokeSync` 切回主线程
- **注意**: `Frameless: true` 会移除窗口关闭按钮，应使用标准窗口
- `ExecJS` 对外部 URL 无效（`runtimeLoaded` 永不 true），需调用 `HandleMessage("wails:runtime:ready")` 强制设为 true
- 已移除自定义脚本功能，改为硬编码注入批量删除脚本（`batchDeleteScript` 常量）
- WebView 超时销毁机制：隐藏超过 5 分钟后自动销毁窗口，下次唤起时从头创建新版 WebView。关键实现：`forceClose atomic.Bool` 控制 RegisterHook 是否跳过取消逻辑、`makeMainWindow()` 工厂函数抽取窗口创建逻辑、`mainWindow` 指针在 `setupHotkey` 中通过 `*mainWin = win` 重新赋值
