## Overview

A standard Wails project follows a predictable layout. This guide walks through
each directory and file you will encounter.

## Root Layout

```
myapp/
├── main.go
├── Taskfile.yml
├── config.yml
├── frontend/
├── build/
└── bin/
```

## main.go

The main entry point. This is where you define your application, register
services, and call `Run()`.

```go
package main

import (
    "github.com/wailsapp/wails/v3/pkg/application"
)

func main() {
    app := application.New(application.Options{})
    app.Run()
}
```

## frontend/

Contains the web-based UI layer. Wails serves this during development and
bundles it into the final binary.

| Path                | Purpose                                      |
|---------------------|----------------------------------------------|
| `index.html`        | HTML entry point                             |
| `src/`              | Application source code                      |
| `src/main.js`       | JavaScript entry point                       |
| `public/`           | Static assets copied verbatim to the build   |
| `dist/`             | Built output (generated by Vite)             |
| `bindings/`         | Auto-generated Go-to-JS bindings             |
| `package.json`      | Node.js dependencies and scripts             |
| `vite.config.ts`    | Vite configuration                           |
| `tsconfig.json`     | TypeScript configuration                     |

## build/

Platform-specific resources and build configuration.

| Path            | Purpose                                      |
|-----------------|----------------------------------------------|
| `appicon.png`   | Application icon (used across platforms)     |
| `darwin/`       | macOS-specific resources (plist, entitlements, icns) |
| `windows/`      | Windows-specific resources (manifest, icon, rc) |
| `linux/`        | Linux-specific resources (desktop files, icons) |
| `android/`      | Android-specific resources                   |
| `ios/`          | iOS-specific resources                       |
| `docker/`       | Docker build files                           |

## bin/

Output directory for compiled binaries. Created automatically during builds.
The structure inside varies by platform:

```
bin/
├── myapp-darwin-arm64/
├── myapp-linux-amd64/
└── myapp-windows-amd64/
```

## Taskfile.yml

Task runner configuration. Wails uses [Task](https://taskfile.dev/) as its
build system. The default Taskfile provides common tasks like `build`, `dev`,
and `package`.

```yaml
version: "3"

tasks:
  build:
    cmds:
      - wails3 build
  dev:
    cmds:
      - wails3 dev
```

## config.yml

Runtime configuration for the Wails CLI. Controls build options, dev server
settings, and platform overrides.

```yaml
name: myapp
outputfilename: myapp
frontend:
  install: npm install
  build: npm run build
  dev: npm run dev
```
